# Shipping Rules

Read first. Write second. Verify third.

## Before

- Read before write: open sources, call sites, tests, schemas, and consumers.
- Name like the next reader is hostile: if you cannot name it cleanly, you do
  not understand it yet.

## During

- Smallest unit that works: one purpose per unit, testable in isolation.
- Delete more than you add: code and claims are liabilities unless they earn
  their weight.

## After

- Evidence over intuition: trust nothing unverified.
- Failure is the spec: root cause, blast radius, security path, and undo plan
  must be explicit.

## Brain Rule

No source, no claim. No approval and rollback, no mutation. No verification
path, no release.
