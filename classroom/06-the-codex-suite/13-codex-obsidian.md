# 🗂️ Codex Obsidian

> Build a wiki vault that Codex can research into, save findings to, and answer questions from with citations. Your research stops evaporating between sessions.

> 🎬 **Watch:** [The AI Second Brain That Actually Remembers Everything](https://youtu.be/I2hXQ-_qzRk)

![](https://cdn.jsdelivr.net/gh/AI-Marketing-Hub/classroom-assets@cfc5a1e99148d256bff96912f16ee28322523f56/anim-real-codex-obsidian.gif)

### What codex-obsidian does

codex-obsidian is the Codex port of the Hub's Obsidian companion. The core loop: scaffold a vault, pull source material in with research, save an answer or session as a structured note, then query and retrieve from what you've saved with citations. It also lints for orphan pages and broken links, and builds visual maps and database views.

### Install it

```bash
codex plugin marketplace add AI-Marketing-Hub/codex-obsidian
codex plugin add codex-obsidian@ai-marketing-hub-codex-obsidian
```

### Run it

Most AI work is throwaway. You get a great answer, close the window, and lose it. codex-obsidian fixes that. Every audit, research dive, and campaign note lands in a vault you own, in plain Markdown, cross-linked and searchable. Ask a follow-up next week and Codex reads the vault first, then answers from your own knowledge with citations.

Save the current work into your vault:

```text
Save this conversation as a structured note in my Obsidian vault.
Pick the right note type, add frontmatter, file it in the correct
folder, and update the index and log.
```

Query what you already know:

```text
Based on my Obsidian vault, what do I already know about my
competitor's pricing strategy? Synthesize with citations to the
source notes.
```

### Same as the Claude version

The standout is the research loop. Point it at a topic, it runs searches, synthesizes findings, and files structured pages straight into your vault. Then query that knowledge later. Build it once, compound it forever. Same multi-writer-safe pattern as Claude + Obsidian. For the full setup and the sub-skills, see the product course.
